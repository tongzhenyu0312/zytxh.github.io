let

let是ES6新增的一种声明变量的方式,它有许多新的特性: 只能在声明命令所在的代码块使用,不存在变量提升,却有暂时性死区,并且不可重复声明等..

1. 只能在自己的代码块(块级作用域)里用

很有意思,let声明的变量只能在其所在的代码块中使用,在ES5中可以随便使用,这样改的好处可以在for循环里体现出来…

for循环:

为什么呢? 就是因为ES6中{}形成的块级作用域中使用let形成了封闭作用域,能够阻止ES5中常会出现的变量的污染(即是指变量的覆盖与泄露)。

在这里,明确几点:

1.块级作用域与函数形成的私有作用域并不等价
2.函数作用域内变量完全隔离与外界,所产生的闭包会保护内部的私有变量;
3.而块级作用域只有配合上let,才能给let声明的那个变量产生类似于私有作用域的效果 2

只要块级作用域内存在let命令，它所声明的变量就“绑定”（binding）这个区域，不再受外部的影响。
ES6 明确规定，如果区块中存在let和const命令，这个区块对这些命令声明的变量，从一开始就形成了封闭作用域。凡是在声明之前就使用这些变量，就会报错。

块级作用域

1. 定义:

一个{}便形成了一个块级作用域,let命令会使这个块级作用域变成封闭作用域,而有以上所有特性;块级作用域中var命令声明的变量可以看成是处在上一级作用域中的。


2. 块级作用域中的函数声明

这里是因为浏览器环境下的es6的函数声明有如下规则:

  • 函数声明类似于var,会提升到全局作用域或者函数作用域的头部
  • 允许在块级作用域内声明,且会提升到块级作用域头部

不同的环境下的es6对于函数的声明有不同的规则,因此,我们在块级作用域内应该尽量使用函数表达式,而非函数声明式



const

const同let和var一样,是一种声明命令

1. 定义:

const声明的变量是不可被重新赋值的,const声明的特性与let一致


2. 本质:

const声明的变量在本质上保存的是指向值得指针,当然这在面对值类型的时候是没什么用场的,而当变量赋值的是一个引用数据类型或者说是复合数据类型的时候,就有意思了。

全局对象与顶层对象

顶层对象在浏览器环境下是window,在node环境下是global,将顶层对象与全局对象挂钩,是被认为Javascript最大的败笔之一,原因有下:

  • 无法在编译时就对未声明的变量抛错,只能在运行时发现
  • 写错了单词…
  • 顶级对象本来就不应该给写的权限,不利于模块化的构建
  • 顶级对象是有着实体含义的,比如浏览器,给它又赋予全局对象的概念不好。

因为这些原因,ES6中通过let const class声明的全局变量都脱离顶层对象



global

顶层对象本身也有一堆问题,比如浏览器的顶层对象是window/self,web worker里面是self,node里面是global

为了统一顶层对象,有两种做法:

  • 函数判断
  • 将所有环境的顶层变量都换成global,垫片库system.global模拟了这个提案，可以在所有环境拿到global。
