ECMAscript6 2015年6月发布,可以编写复杂的大型应用程序的企业级开发语言。









1. Javascript 与 ES

  • 1996年11月,Netscape向ECMA提交将Javascript作为一门浏览器脚本语言的提案;

  • 1997年Js正式面世,由于java和javascript已经被注册且需要体现ecma的功劳,取名ECMAScript;



2. ES6 与 ES2015

  • ES从1997年发布ES 1.0至ES 3.0,初学者从3.0开始学;

  • ES 4.0太激进,变化巨大,浏览器厂商特别不满意,联合抵抗;

  • 迫于压力发布的ES 3.1改动较小,即是后来的ES5。

  • ES 4.0成为后来的ES6,在2015年6月发布,又叫做ES 2015。

  • ECMA规定每年发布一个版本,现在已经出到ES 2018…       :hugs:



3. 部署:

  • 浏览器对于ES6的支持已经达到90%
    检查浏览器支持度
  • node对于ES6的支持度更好,因为node是服务器运行环境(runtime)



4. Babel


babel是一个转码器,其下的多个模块对于转码简直不遗余力(常被用来转码ES6)。下面来讲几个babel全家桶里面的工具或模块,有了它们就可以随便放心的写ES6了:
.babelrc
babel-cli



babelrc


.babelrc是babel的配置项,规定了babel使用哪种规则进行转码。使用babel必须要配,但是
现在可以使用babel-preset-env而无需添加配置项。




工具babel-cli

babel-cli自带babel命令用于转码,可以将代码根据.babelrc的配置规则进行转码
好处: 手动的去转码

source map

优点: 简单来说,生产环境的代码肯定是编译或者压缩过的,这样线上报错定位不到;可以使用工具产生sourc map文件,它与源文件处在同一个文件夹内,存储着源代码的位置信息,方便错误定位。

babel-node

babel-node是babel-cli中的命令之一,能够产生node的repl交互式运行(read-eval-printf-loop)环境。
优点: 应该是类似于node的功能,也可以使用配置package.json和命令行启动的方式来使用。







babel-register

babel-register可以对require命令导入的文件进行转码,注意源文件并不会被转码

babel-core

引入babel-core就可以使用babel的api啦~



babel-polyfill

babel默认只能转码新的JS语法(syntax),而无法转码新的API,例如iterator、map等,这是就需要用到babel-polyfill。Babel 默认不转码的 API 非常多，详细清单可以查看babel-plugin-transform-runtime模块的definitions.js文件。




polyfill与shim

polyfill是浏览器端的shim
shim指的是一个库,用旧环境的API实现全新的API
polyfill就是一系列的代码或者插件，它为开发者提供的技术特性，都是希望浏览器本就应该原生支持的，并且抹平了 api 之间的使用差异。典型地像检测浏览器是否支持某一个 api，如果不支持就加载一个 polyfill，这样就可以让开发者在任何情况下无缝的使用那些 api 了。



浏览器

babel其实是可以运行在浏览器环境下的,babel 6.0后虽然不再支持了,但是可以使用babel-standalone来实现浏览器的实时转码(比较消耗性能)
